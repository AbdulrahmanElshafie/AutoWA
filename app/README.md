# WhatsApp Automation Core (WA Controller)

## Overview

This module implements the **core WhatsApp Web automation logic** for the application.
It is responsible for controlling the desktop, interacting with WhatsApp Web, managing contacts, and sending messages or documents in a **human-like, profile-driven manner**.

The automation is built on top of **image-based GUI control** using `pyautogui`, combined with randomized timing profiles to reduce bot-like behavior.

This README focuses exclusively on the **WhatsApp controller layer**, independent of the GUI layer or higher-level business logic.

---

## Purpose

The WhatsApp Automation Core is designed to:

- Open WhatsApp Web in supported browsers
- Navigate WhatsApp UI reliably using image recognition
- Add, search, and delete contacts dynamically
- Send text messages and PDF documents
- Behave in a human-like way using randomized timing profiles

It acts as a **reusable automation engine** that can be driven by any interface (GUI, CLI, scheduler, etc.).

---

## High-Level Architecture

```text
helpers.py        → Timing profiles for typing & movements speed, and waiting gaps 
    ↓
Controller.py    → Generic desktop automation (pyautogui)
    ↓
WAController.py   → WhatsApp-specific workflows
```

Each layer has a single responsibility and can be extended independently.

---

## File Responsibilities

### 1. `Controller.py` – GUI Automation Layer

This file provides a **generic desktop automation controller** built on top of `pyautogui`.

#### Responsibilities

- Locate UI elements using screenshots
- Click, type, paste, and press keys
- Wait for UI conditions to appear
- Apply timing profiles to every interaction

#### Key Concepts

- **Image-based automation**: Every action depends on screenshots in `/assets`
- **Profile-driven timing**: All delays are generated by `TimeConroller`

This file contains **no WhatsApp logic** and can be reused for other desktop automation tasks.

---

## 2. `WAController.py` – WhatsApp Logic Layer

This file contains **all WhatsApp-specific workflows** built on top of `GuiController`.

#### Responsibilities

- Open and close WhatsApp Web
- Handle browser-specific UI differences (Chrome / Edge)
- Reset zoom and maximize tabs for UI consistency
- Add contacts or search existing numbers
- Delete contacts after sending (when needed)
- Generate randomized message templates
- Send messages with or without document attachments

#### Supported Workflows

- Message-only sending
- Message + PDF document sending
- Multi-browser account rotation

All WhatsApp UI interactions are implemented using **image recognition**, not DOM access.

---

### 3. `helpers.py` – Timing Layer

This file acts as the **actions speed manager (e.g. typing, moving, waiting)** with an error margin to add a randomness factor to mimic human interactions.

#### Responsibilities

- Select a typing and moving speed
- Select a waiting behavior

---

## Assets Directory

The `assets/` folder contains **all UI reference images** used for automation.

### Key Notes

- Each image represents a WhatsApp UI element
- Browser-specific icons are separated (Chrome vs Edge)
- Image names are treated as **API contracts** — renaming them will break automation

Example:

```text
Msg Bar.PNG
```

---

## Human-Like Behavior Design

The system avoids bot-like patterns by:

- Randomized typing speed
- Randomized mouse movement delays

All timing is controlled via **profiles**, not hardcoded values.

---

## Requirements

### Software

- Python 3.9+
- Google Chrome (required)
- Microsoft Edge (optional, second account)
- WhatsApp Web logged in on all used browsers

### Python Dependencies

- [pyautogui](https://pypi.org/project/PyAutoGUI/)
- [pyperclip](https://pypi.org/project/pyperclip/)
- [opencv-python](https://pypi.org/project/opencv-python/)

### System Requirements

- Stable screen resolution
- UI scaling set to 100%
- No user interaction during execution

## Extending the WA Controller

Common extension points:

- Add new message templates
- Support new document types
- Add new browsers/accounts
- Improve image detection reliability
- Add retry logic per contact
- Add dry-run or simulation mode

The layered design allows extending WhatsApp behavior **without touching** the GUI automation layer.

---

## Important Notes

- This module depends entirely on UI stability
- Any WhatsApp UI update may require asset updates
- Do not multitask while automation is running
- Always test with small datasets first
